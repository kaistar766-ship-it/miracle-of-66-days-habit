# 할 일 관리 앱

빌드 도구, 프레임워크, 외부 라이브러리 없이 순수 HTML/CSS/JavaScript로 만든 오늘 하루용 할 일 관리 앱입니다. 완료율을 매일 채워가는 "연속 달성 챌린지(streak)" 기능이 핵심입니다.

## 실행 방법

`index.html`을 더블클릭해서 여세요. `file://` 프로토콜에서 바로 동작하도록 만들어져 있어 서버나 설치 과정이 필요 없습니다.

## 기능

- **여러 프로필**: 앱을 열면 넷플릭스처럼 프로필 선택 화면이 먼저 나옵니다. 이름(+ 선택적으로 사진, 없으면 이니셜)으로 프로필을 추가하고, 프로필별로 할 일·챌린지·카테고리가 완전히 분리되어 저장됩니다. "프로필 관리"를 누르면 각 프로필에 삭제(×) 버튼과 수정(✎) 버튼이 나타나 이름·사진·색상을 바꾸거나(최소 1개는 유지, 삭제 불가) 필요 없는 프로필을 지울 수 있습니다. 프로필 아바타는 선택한 색상이 테두리로 감쌉니다. 메인 화면 헤더의 "프로필 전환"으로 언제든 선택 화면으로 돌아갈 수 있습니다.
- **BGM 플레이어 + 날짜/시간/날씨**: 화면 맨 위에 재생 버튼과 볼륨 슬라이더가 있고, 프로필을 선택해 앱에 들어가는 순간 브라우저에서 직접 합성한 밝고 경쾌한 아르페지오(코드 진행 C-G-Am-F, 128bpm, 저작권 문제 없는 코드 생성 음원)가 자동으로 흐르기 시작합니다. 버튼으로 언제든 끄고 켤 수 있습니다. 바로 옆에는 현재 날짜/시간이 항상 표시되고, "🌤️ 날씨 보기"를 직접 눌러야만 위치 권한을 요청해 Open-Meteo(무료, API 키 불필요)로 현재 날씨/기온을 보여줍니다 — 앱을 열 때마다 자동으로 위치 권한을 요청하지 않도록 일부러 클릭해야만 동작하게 만들었습니다.
- **달성한 날의 히트맵 도장**: 챌린지 조건을 달성한 날은 히트맵 칸이 단색 대신 도장 그래픽(원형/별/리본/하트 중 선택)으로 표시됩니다. 모양·색상·도장 안에 들어가는 문구(최대 12자)를 챌린지 "⚙ 설정"에서 직접 고를 수 있습니다.
- **할 일 CRUD**: 텍스트 + 카테고리(업무/개인/공부)로 추가, Enter 키로도 추가 가능, 인라인 수정(Enter 저장/Esc 취소), 삭제 후 5초간 실행 취소 가능
- **완료 체크**: 체크박스로 완료 토글, 완료 항목은 취소선 표시 및 목록 하단으로 이동
- **오늘 기준 목록**: 오늘 생성한 할 일만 표시
- **카테고리 관리**: 기본 제공되는 업무/개인/공부 외에 이름과 색상을 직접 정해 카테고리를 추가할 수 있습니다. 사용자가 추가한 카테고리는 삭제도 가능하며(기본 3종은 삭제 불가), 삭제 시 그 카테고리를 쓰던 할 일은 자동으로 "업무"로 이동합니다.
- **카테고리 필터**: 전체 + 등록된 카테고리별 탭으로 목록만 필터링 (진행률 계산에는 영향 없음)
- **진행률**: 전체 완료율 + 카테고리별 미니 진행률을 실시간으로 표시
- **연속 달성 챌린지**: 목표일 수·달성 기준(전부 완료 / 최소 N개 완료)을 직접 설정 가능. 오늘 조건을 달성하면 연속일이 올라가고, 체크를 해제해 다시 미달이 되면 자동으로 롤백됩니다. 목표일 수에 맞춘 히트맵과 최고 기록도 함께 보여줍니다.
- **데이터 백업(내보내기/가져오기)**: 할 일과 챌린지 기록을 JSON 파일로 내보내고, 다른 기기/브라우저에서 그 파일을 가져와 복원할 수 있습니다. 가져오기는 현재 기기의 데이터를 덮어쓰므로 확인 절차를 거칩니다.

## 데이터 저장 방식과 한계

모든 데이터는 브라우저의 `localStorage`에 저장됩니다.

- `todoapp.profiles` — 프로필 목록(이름/색상/사진). 프로필과 무관하게 하나만 존재합니다.
- `todoapp.todos.<프로필id>` — 그 프로필의 할 일 목록
- `todoapp.challenge.<프로필id>` — 그 프로필의 챌린지(연속 달성) 상태
- `todoapp.lastCategory.<프로필id>` — 그 프로필이 마지막으로 사용한 카테고리
- `todoapp.categories.<프로필id>` — 그 프로필의 카테고리 목록(기본 3종 + 추가한 카테고리)

(프로필 개념 도입 이전 버전에서 쓰던 데이터가 남아 있다면 "기본 프로필"로 자동 편입되어 접미사 없이 그대로 사용됩니다.)

이로 인한 한계:

- **브라우저/기기 종속**: 데이터는 지금 열어 둔 브라우저의 로컬 저장소에만 남습니다. 다른 브라우저, 다른 기기, 시크릿/프라이빗 모드에서는 보이지 않으며, 자동 동기화 기능은 없습니다. 여러 기기에서 같은 기록을 이어가려면 "데이터 백업" 영역의 내보내기/가져오기를 수동으로 사용해야 합니다.
- **브라우저 데이터 삭제 시 유실**: "인터넷 사용 기록 삭제" 등으로 사이트 데이터/쿠키를 지우면 저장된 할 일과 챌린지 기록도 함께 사라집니다. 중요한 기록은 주기적으로 내보내기 해 두는 것을 권장합니다.
- **저장 공간 제한**: `localStorage`는 브라우저별로 용량 제한이 있습니다. 저장에 실패하면 화면 상단에 안내 토스트가 표시됩니다.
- **날씨는 인터넷 연결과 위치 권한이 필요**: BGM은 완전히 오프라인으로 동작하지만, 날씨/기온 정보는 "🌤️ 날씨 보기"를 눌렀을 때 인터넷 연결과 브라우저의 위치 권한 허용이 있어야 표시됩니다. 오프라인이거나 권한을 거부하면 버튼이 다시 "🌤️ 날씨 보기"로 돌아가며, 날짜/시간만 계속 표시됩니다.
